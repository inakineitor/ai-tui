# Fix QueuedMessage Build Warning

## Problem

During build (`pnpm build`), tsdown shows `[MISSING_EXPORT]` warnings:

```
src/routes/session/components/message-list/components/message/index.d.ts (3:9) 
[MISSING_EXPORT] Warning: "QueuedMessage" is not exported by "src/hooks/use-message-queue.d.ts"
```

This occurs because:
1. `QueuedMessage` and `FileUIPart` types are exported from `use-message-queue.ts`
2. Internal components import these types: `import type { QueuedMessage } from "#hooks/use-message-queue.js"`
3. These types are NOT exported from the main `index.ts` entry point
4. When tsdown bundles declarations, it can't resolve the internal type imports

## Solution

Export `QueuedMessage` and `FileUIPart` from the main package entry point (`packages/core/src/index.ts`).

## Files to Modify

- `packages/core/src/index.ts` - Add exports for `QueuedMessage` and `FileUIPart`

## Implementation

Add to `packages/core/src/index.ts`:

```typescript
// Message queue types (used by QueuedUserMessage component)
export type { FileUIPart, QueuedMessage } from "./hooks/use-message-queue.js";
```

## Verification

1. Run `pnpm build` - should complete without `[MISSING_EXPORT]` warnings
2. Run `pnpm check-types` in `packages/core` - should pass
3. Verify the types appear in `dist/index.d.mts`
