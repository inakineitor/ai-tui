## Problem

Both `@ai-tui/core` and `@ai-tui/components` packages have broken builds because the tsdown output files don't match what's configured in `package.json`.

**Expected files (per package.json):**
- `dist/index.cjs`
- `dist/index.mjs`
- `dist/index.d.mts`
- `dist/index.d.cts`

**Actual files (tsdown output):**
- `dist/..cjs`
- `dist/..mjs`
- `dist/..d.mts`
- `dist/..d.cts`

## Root Cause

The `tsdown.config.ts` files use `"."` as the entry key:

```typescript
entry: {
  ".": "src/index.tsx",
}
```

The entry key becomes the output filename, so `"."` produces files like `..cjs` (dot + dot + extension).

## Solution

Change the entry key from `"."` to `"index"` in both packages:

```typescript
entry: {
  "index": "src/index.tsx",
}
```

## Files to Modify

1. `packages/core/tsdown.config.ts` - Change entry key from `"."` to `"index"`
2. `packages/components/tsdown.config.ts` - Change entry key from `"."` to `"index"`

## Verification

2. Verify the dist folders contain correctly named files:

   - `packages/components/dist/index.mjs`

3. Run type checking: `pnpm typecheck`
